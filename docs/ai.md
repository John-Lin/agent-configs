# AI Tooling

## Shared instructions (AGENTS.md)

All three agents read one canonical instruction file, generated from
`agents-md/AGENTS.base.md` (+ optional gitignored `agents-md/AGENTS.personal.md`):

- Canonical file: `~/.pi/agent/AGENTS.md` (a real file â€” pi owns it)
- `~/.claude/CLAUDE.md` â†’ symlink â†’ `~/.pi/agent/AGENTS.md` (Claude Code)
- `~/.config/opencode/AGENTS.md` â†’ symlink â†’ `~/.pi/agent/AGENTS.md` (OpenCode)

Any of `make sync-claude` / `sync-opencode` / `sync-pi` regenerates the canonical
file first, so it always exists before the symlinks are created. If the canonical
file has drifted from the repo, the sync stops; run `make sync-pi-force` to replace it.

## Claude Code

Claude Code config lives in `claude/`.

Install:

```bash
make sync-claude
```

If local Claude files already contain unmanaged contents, the sync stops instead of overwriting them.
Use `make sync-claude-force` only when you explicitly want to replace local contents.

Detailed Claude setup and personal overrides are documented in `claude/README.md`.

## OpenCode

OpenCode agents live in `opencode/agents/`.
The generated `~/.config/opencode/opencode.json` comes from the Jsonnet source under `jsonnet/`.
Global instructions come from `~/.config/opencode/AGENTS.md`, which symlinks to the canonical pi file above.

Install:

```bash
make sync-opencode
```

If `~/.config/opencode/opencode.json`, `~/.config/opencode/agents`, or `~/.config/opencode/AGENTS.md` already contains unmanaged contents, the sync stops.
Use `make sync-opencode-force` to replace them explicitly.

See `jsonnet/README.md` for the config source, supported models, and work-overlay flow.

## Claude Settings

`~/.claude/settings.json` is generated by merging:

- `claude/claude_settings.json.template`
- `claude/claude_settings.personal.json`

Personal settings take precedence over the template.

## pi

Install:

```bash
make sync-pi
```

pi owns the canonical `~/.pi/agent/AGENTS.md` (see *Shared instructions* above). `make sync-pi` regenerates it and injects shared packages into `~/.pi/agent/settings.json`, keeping personal settings (provider, model, etc.) untouched.

## Skills

Skills are tool-neutral and shared via one canonical location, the same way as the
instructions. The source lives in the repo-root `skills/` package; `make sync-skills`
stows each skill to `~/.agents/skills/<name>/SKILL.md`.

Discovery per tool:

- **OpenCode** and **pi** read `~/.agents/skills/` natively â€” no symlink needed.
- **Claude Code** reads only `~/.claude/skills/`, so `make sync-claude` points
  `~/.claude/skills â†’ ~/.agents/skills`.

Install:

```bash
make sync-skills
```

Tracked shared skills:

- `architecture-diagram` - standalone architecture diagrams as HTML/SVG
- `find-docs` - Context7-powered documentation lookup
- `gh-cli` - GitHub CLI workflow reference
- `test-driven-development` - TDD workflow
- `omarchy` - Linux desktop/window manager customization (a machine-local pointer;
  linked only on machines where omarchy is installed)
